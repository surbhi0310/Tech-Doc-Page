Single Page Application(SPA)

These are special web apps in which we only have one html page that is rendered.Any content we have to render or any interactivity, even going from one page to another all handled by js only.Interesting part is the app doesn't refresh while interacting or moving from one pg to another.

When we write code in plain Vanilla js(normal js) the problem with this exist is- repetitive code; it's very hard to manage

What is React?

React is a JS library, it comes with a bare minimum set of functionalities using which we can make mordern SPA with ease. It doesn't bring a lot of conventions or baggage with it , and gives just the required functionalities for an SPA.

This is a problem of react also, that for a lot of things we have rely on third party Solutions/libraries.

    Features of React are -
  1. React uses a Virtual DOM to optimize UI rendering. Instead of updating the entire real DOM directly, React
  2. React follows a component-based approach, where the UI is broken down into reusable components.(can be functional & class based
  3. React uses JSX, a syntax extension that allows developers to write HTML inside JavaScript.
  4. React uses one-way data binding, meaning data flows in a single direction from parent components to child components via props.
  5. State Management-React manages component state efficiently using the useState hook (for functional components) or this.state (for class components). State allows dynamic updates without reloading the page.
  6. Hooks allow functional components to use state and lifecycle features without needing class components.
  7. React provides React Router for managing navigation in single-page applications (SPAs). It enables dynamic routing without requiring full-page reloads.

Note: Components follow DRY Principle(do not repeat same piece of ui) heavily

Let's discuss project structure:-

Let's go over every folder and file one by one:

  • package.json: It is the most important files which contain metadata about the project. It tells important details of the project like name of project, version of project(self configurable), scripts, it also tells the dependencies which our project has. For eg: dev script written is executed when we execute npm run dev.
  • node_modules: This folder contains the downloaded dependency codes. For eg: we have eslint react etc as dependency of the project
  • pakage-lock.json: Now the dependencies we installed also have inner denpendencies, so this file keeps the version details of al the dependencies and inner dependencies.So, if we loose node_modules we can download the same version of everything because of this.
  • gitignore:This file contains all the files and folders that we don't wat to upload on github once we upload the project on github.
  • eslintrc:This files keeps the eslint configuration.
  • vite.config.js:This file keeps the react configuration.
  • public:Any static file like pdfs, images etc goes here.
  • src:Your Actual code is present.
JSX

JSX(extension to js) is an extention to XML, that's why the syntax of JSX looks extremely similar to XML and HTML (HTML is also a derivative of XML only). Meta prepared JSX for making the UI implementation more easy, because using plain JS functions that React and other technologies provide can make the code unreadable. And with JSX, because it is a very HTML like syntax it becomes easy.

Features

  • Looks and feels like HTML.
  • Has some minor differences compared to HTML.
  • Writing complex UI logic is easy with JSX.
  • Get's easily embedded with any JS code as well i.e in the same file we can write JS and JSX together.

JSX is not directly rendered as HTML by React; instead, it gets compiled into JavaScript objects representing virtual DOM elements. These objects are later used by React to efficiently update the real DOM.

Meta didn't intent to get JSX compatibility added in the browser or in EcmaScript but they intent that there should be a layer of transpilation that can convert JSX to normal JS code.

Transpilation

It is the process of converting src code from one high-level programming languagr to another. We use transpiler like Babel to convert JSX to compatible JS.To use JSX we need to make aa file with .jsx extension.

Now, In a react code base if we have a function that returns some kind of JSX, we will call that a Component or more specifically a Functional Component because this component has been made using functions. We can make components using class also.

Defining a function(function App(){return ...})
Calling a function <"App /">(ignore inverted commas)
Note: Name of a component must start with a capital letter else it will not work properly.The custom component must be in Pascalcase(like CustomComponent) and inbuilt HTML element based components like div, h1, h2 etc should be in lowercase.
