Introduction

CSS (Cascading Style Sheets) is used to style and layout web pages.

    Ways to Add CSS -

  • Inline CSS :- Applied directly within an HTML element using the style attribute.
  • Internal CSS :- Placed within a style tag inside the head section of an HTML document.
  • External CSS :- Linked via a link tag to an external CSS file.
  • Cascading Order :- Order of priority: Inline > Internal > External.

    • Comments in CSS -

  • Single-line Comment:- /*This is a single-line comment */
  • Multi-line Comment:- /* This is a multi-line comment. It can span multiple line */

Selectors
  • Tagname Selector :- Selects all elements of a specified tag.
    Ex: p { }
  • ID Selector :- Selects a single element with a specific ID.
    Ex: #unique-id { }
  • Class Selector :- Selects all elements with a specific class.
    Ex: .class-name { }
  • Universal Selector :- Selects all elements.
    Ex: * { }
  • Grouping Selector :- Groups multiple selectors to apply the same styles.
    Ex: h1, h2, p { }

Box Model
The CSS box model is a container that contains multiple properties, including borders, margins, padding, and the content itself.

In CSS, whenever we visualize any HTML element,we nshould visualise it in the form of a rectangular box. The rectangular box of HTML elements has got 4 parts.

  • Content
  • Padding - padding: 12px 4px 3px 20px (top,right,bottom,left)
  • Border -

    border-style: solid(none, solid, dashed, dotted, groove)
    border-width: 2px(3pt,4cm,thick,thin,medium)
    border-color: black
    border-radius: 20px
    Shorthand property -
    border: 5px solid black (border-size border-style border-color)

  • Margin - margin: 10px 20px 12px 13px margin:auto(whatever the space left in the parent element it take it as margin.)
Box model -
1. Content-box(by default) -> In this width = content box only
2. Border-box -> In this width = content box + padding + border

box-sizing: content-box(by default)/border-box

Value:

  • percentage(_% of the parent element)
  • initial(default value)
  • inherit
  • auto
For block elements like div, if we give auto it expands the width to fill the entire hortizontal space till the corner.
For inline elements like span, if we giveauto it expand width to the content width only.

CSS Units

CSS Units define the metric using which we put the size and length and dimensions of our HTML Elements.- Two types of css units are -
1. Absolute Units
These are the fixed-length units, they stay unchanged regardless of the device configuration.

    The Absolute unit types consist of -
  • Pixels(px)
  • centimeters (cm) - 1cm = 37.8px(roughly)
  • Millimetres(mm) - 1mm = 3.78px(roughly)
  • inches(in) - 1in = 96px = 2.54cm(roughly)
  • Points(pt) - 1pt = 1.3333px(roughly)
  • Picas(pc) - 1pc = 16px (roughly)

2. Relative Units
Relative units are good for styling the responsive site because they scale relative to the window size or the parent (depending on the unit).
    The Relative unit types consist of -
  • percentage(%) - relative to the parent element's value.
  • parent element's font size(em) - relative to the font size of the parent element.
  • root element's font size(rem) - relative to the font- size of the root element.
  • viewport height(vh) - relative to the height of the viewport. (1vh = 1%)
  • viewport width(vw) - relative to the width of the viewport. (1vw = 1%)
  • viewport minimum(vmin) - relative to the smaller dimension of the viewport
  • viewport maximum(vmax) - relative to the larger dimension of the viewport. 1vmax = 1%
  • character(ch)
  • X-height(ex)

DOM (Docment Object Model)

Internals of CSS (how html, css, js code runs on browser whenever loads a page)

    There are 5 steps / CRP(Critical Rendering Path)
  1. Parse HTML:-
    • To parse the HTML, browser first of all tokenizes(convert html tags to html elements) our HTML.
    • Browser creates html elements with these tokens and then connect them in a tree like structure to create DOM.
  2. CSS Parsing:-
    • Browser parses CSS in a similar way, tokenize it and create a tree like structure called as CSSOM.
    • Browser executes an internal algorithm called as SELECTOR MATCHING(SM).
    • SM decides the final set of styles which will be applied to any element.
  3. Render Tree:-
    • Final tree created by combination og DOM & CSSOM.
    • It contains only those html elements/nodes which are finally going to be rendered.
  4. Layout(Reflow Stage):-
    • Browser uses the render tree & then starts calculating measurements & actual position on the page where the elements need to come.
  5. Painting Step

Display and Visibility

Display Property -

  1. Block (Creates a block level element, starts onto a new line, expands horizontally if width is not defined)
  2. Inline (Creates an inline level element, box will remain on the same line, Vertical Padding, margins, and borders won't take effect)
  3. Inline-block (combines characteristics of both inline and block elements, allows to set a width and height, the top and bottom margins/paddings are respected)
  4. None (used to make an element invisible, the element will not take any space in the document.)
  5. Table
  6. Inline-table
  7. Flex
